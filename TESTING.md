# Инструкция по тестированию админ-панели

## 🚀 Быстрый старт

Админ-панель настроена для работы с **мок-данными**, поэтому вы можете протестировать её без запуска backend.

### 1. Установка зависимостей

```bash
cd admin
npm install
```

### 2. Запуск приложения

```bash
npm start
```

Приложение откроется на `http://localhost:3000`

### 3. Вход в систему

Используйте тестовые учетные данные:

- **Телефон:** `+79999999999`
- **Пароль:** `admin123`

## 🧪 Сценарии тестирования

### Сценарий 1: Работа с заказами
1. Войдите в систему
2. Перейдите в "Заказы"
3. Отфильтруйте заказы по статусу "Ожидает"
4. Кликните на заказ для просмотра деталей
5. Назначьте исполнителя на заказ
6. Измените статус заказа на "Выполнен"

### Сценарий 2: Управление пользователями
1. Перейдите в "Пользователи"
2. Отфильтруйте по роли "Исполнитель"
3. Просмотрите профиль исполнителя
4. Верифицируйте исполнителя
5. Заблокируйте пользователя

### Сценарий 3: Обработка обращений
1. Перейдите в "Поддержка"
2. Откройте тикет со статусом "Открыт"
3. Напишите ответ на обращение
4. Измените статус на "Закрыт"

### Сценарий 4: Работа с платежами
1. Перейдите в "Платежи"
2. Просмотрите список платежей
3. Отфильтруйте по статусу "Завершен"
4. Выполните возврат платежа

## 🔧 Технические детали

### Мок-данные
Все данные хранятся в файлах:
- `admin/src/services/mockData.ts` - тестовые данные
- `admin/src/services/mockApi.ts` - мок-API сервис

### Задержка сети
Мок-API имитирует задержку сети (500ms) для реалистичности.

### Авторизация
Токен сохраняется в `localStorage` и проверяется при каждом переходе.

## 🐛 Известные ограничения

1. **Данные не сохраняются** - при перезагрузке страницы все изменения сбрасываются
2. **Пагинация** - работает только визуально, все данные загружаются сразу
3. **Поиск** - не реализован в мок-версии
4. **Загрузка файлов** - не поддерживается

## 🔄 Переключение на реальный backend

Когда backend будет готов, замените в файлах:

```typescript
// Было
import mockApi from '../services/mockApi';

// Станет
import api from '../services/api';
```

И обновите вызовы API согласно реальным endpoint'ам.

## 💡 Советы

1. **Откройте DevTools** (F12) для просмотра логов и ошибок
2. **Проверьте Network** - все запросы должны выполняться мгновенно (мок)
3. **Тестируйте на разных разрешениях** - интерфейс адаптивный
4. **Проверьте localStorage** - там хранится токен авторизации

## 🆘 Проблемы?

Если что-то не работает:
1. Очистите кэш браузера
2. Удалите `node_modules` и выполните `npm install`
3. Проверьте консоль браузера на ошибки
4. Убедитесь, что используете Node.js версии 16+
